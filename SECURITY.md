# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Obsession Tracker, please report it
responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email **security@obsessiontracker.com** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Fix Target**: Within 30 days for critical issues

## Scope

The following are in scope for security reports:

- Obsession Tracker Flutter app (this repository)
- Tracker API server (`server/` directory)
- Data storage and encryption
- Authentication and authorization
- Privacy-related issues (unintentional data exposure)

## Security Architecture

For detailed information about the app's security architecture, encryption
methods, and security scanning tools, see
[docs/SECURITY.md](docs/SECURITY.md).

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | Yes                |
| < Latest| Best effort        |

## Recognition

We appreciate responsible disclosure and will credit security researchers in our
release notes (with permission).
